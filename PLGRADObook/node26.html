Construyendo Árboles con la Asociatividad Correcta

Añadamos el operador - al código de nuestra práctica. Para ello, podemos extender nuestro gramática con una regla de producción:


expression term '+' expression term '-' expression term


que da lugar a un código como el que sigue: 
  expression = ->
    result = term()
    if lookahead and lookahead.type is "+"
      ....
    if lookahead and lookahead.type is "-" 
      match "-" 
      right = expression()
      result =
        type: "-" 
        left: result
        right: right
Cuando le damos como entrada a = 4-2-1 produce el siguiente AST: 
{
  "type": "=",
  "left": {
    "type": "ID",
    "value": "a"
  },
  "right": {
    "type": "-",
    "left": {
      "type": "NUM",
      "value": 4
    },
    "right": {
      "type": "-",
      "left": {
        "type": "NUM",
        "value": 2
      },
      "right": {
        "type": "NUM",
        "value": 1
      }
    }
  }
}
que se corresponde con esta parentización: a = (4 - (2 - 1))

Este árbol no se corresponde con la asociatividad a izquierdas del operador -. Es un árbol que refleja una asociación a derechas (a = 3).
